# Coast Runtime

Este proyecto usa Coasts (host en contenedores) para entornos de desarrollo aislados.
Tus ediciones de código son automáticamente visibles dentro del Coast en ejecución — el sistema de archivos
se comparte entre el host y el contenedor.

## Orientación

Antes de ejecutar cualquier comando de runtime, descubre qué instancia de Coast coincide con tu
directorio de trabajo actual:

```
coast lookup
```

Esto imprime el nombre de la instancia, puertos, URLs y comandos de ejemplo. Usa el nombre de la
instancia de la salida para todos los comandos posteriores.

Si necesitas un contexto más profundo sobre cómo funcionan los Coasts, lee esta documentación:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Ejecución de comandos

Usa `coast exec` para ejecutar comandos dentro del Coast. El shell comienza en la raíz del workspace
(donde está el Coastfile). Primero haz cd al directorio de destino:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

Ejemplos:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Retroalimentación del runtime

Comprueba el estado del servicio:

```
coast ps <instance>
```

Lee los logs del servicio:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Resolución de problemas

Si encuentras errores o comportamientos desconocidos, busca en la documentación de Coast:

```
coast search-docs "mensaje de error o descripción"
```

Esto usa búsqueda semántica — describe el problema en lenguaje natural y encontrará
la documentación relevante.

## Conocimiento del worktree

Cuando empieces a trabajar en un worktree — ya sea que lo hayas creado tú o una herramienta como
Codex, Conductor o T3 Code lo haya creado por ti — comprueba si una instancia de Coast ya está
asignada:

```
coast lookup
```

Si `coast lookup` encuentra una instancia, úsala para todos los comandos de runtime.

Si no devuelve instancias, comprueba qué se está ejecutando actualmente:

```
coast ls
```

Luego pregunta al usuario qué opción prefiere:

**Opción 1 — Crear un nuevo Coast y asignar este worktree:**
```
coast run <new-name>
coast assign <new-name> -w <worktree>
```

**Opción 2 — Reasignar un Coast existente a este worktree:**
```
coast assign <existing-name> -w <worktree>
```

**Opción 3 — Omitir Coast por completo:**
Continúa sin un entorno de runtime. Puedes editar archivos pero no puedes ejecutar pruebas,
compilaciones ni servicios dentro de un contenedor.

El valor `<worktree>` es el nombre de la rama (ejecuta `git branch --show-current`) o
el identificador de worktree mostrado en `coast ls`. Pregunta siempre al usuario antes de crear
o reasignar — no lo hagas automáticamente.

## Reglas

- Ejecuta siempre `coast lookup` antes de tu primer comando de runtime en una sesión.
- No ejecutes servicios directamente en el host. Usa `coast exec` para todas las tareas de runtime.
- Las ediciones de archivos en el host son visibles instantáneamente dentro del Coast. No necesitas
  copiar archivos ni reconstruir después de editar.
- Si `coast lookup` no devuelve instancias, puede que el Coast no esté en ejecución. Sigue el flujo de
  conocimiento del worktree anterior para resolver esto con el usuario.
