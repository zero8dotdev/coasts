# Coast Runtime

Este proyecto usa Coasts — runtimes en contenedores para ejecutar servicios, pruebas
y otros comandos de runtime. El sistema de archivos se comparte entre el host y el
contenedor, por lo que las ediciones de archivos en cualquiera de los dos lados son visibles
para ambos de inmediato.

## Discovery

Antes del primer comando de runtime en una sesión, ejecuta:

  coast lookup

Esto imprime el nombre de la instancia, los puertos y comandos de ejemplo. Usa el nombre
de la instancia de la salida para todos los comandos posteriores.

## What runs where

El sistema de archivos se comparte, así que usa `coast exec` solo para las cosas que necesitan
el runtime del contenedor (bases de datos, servicios, pruebas de integración). Todo lo demás
se ejecuta directamente en el host.

Usa `coast exec` para:
- Pruebas que necesitan servicios en ejecución (pruebas unitarias integradas con servicios o dbs, pruebas de integración, pruebas de API)
- Reinicios de servicios u operaciones de compose
- Cualquier cosa que se comunique con bases de datos, cachés u otros servicios del contenedor

Ejecuta directamente en el host:
- Linting, verificación de tipos, formateo
- Operaciones de Git
- Playwright y pruebas de navegador
- Instalar dependencias del lado del host (npm install, pip install)
- Búsqueda de archivos, generación de código, análisis estático

Ejemplo:

  coast exec <instance> -- sh -c "cd <dir> && npm test"    # needs DB
  npm run lint                                              # host is fine
  npx playwright test                                       # host is fine

## Runtime feedback

  coast ps <instance>
  coast logs <instance> --service <service>
  coast logs <instance> --service <service> --tail 50

## Creating and assigning Coasts

Si `coast lookup` no devuelve ninguna coincidencia, ejecuta `coast ls` para ver qué existe.

Si ya hay un Coast sin asignar en ejecución para este proyecto, prefiere asignarle
tu worktree en lugar de crear uno nuevo:

  coast assign <existing> -w <worktree>

Un Coast ya ocupado también puede reasignarse con `coast assign`, pero primero consúltalo
con el usuario porque eso interrumpirá el slot actual.

Si no hay ningún Coast en ejecución, pregunta al usuario antes de crear uno — los Coasts
pueden consumir mucha memoria:

  coast run <name> -w <worktree>

Un proyecto debe compilarse antes de que se puedan crear instancias. Si `coast run` falla
porque no existe ninguna compilación, ejecuta primero `coast build`.

## Coastfile setup

Si el proyecto todavía no tiene un Coastfile, o si necesitas modificar el
Coastfile, primero lee la documentación del Coastfile:

  coast docs --path coastfiles/README.md

## Rules

- Ejecuta siempre `coast lookup` antes de tu primer comando de runtime en una sesión.
- Usa `coast exec` solo para las cosas que necesitan el runtime del contenedor.
- Ejecuta linting, verificación de tipos, formateo y git directamente en el host.
- Usa `coast docs` o `coast search-docs` antes de hacer suposiciones sobre el comportamiento de Coast.
- No ejecutes servicios directamente en el host cuando el proyecto espera Coast.

# Coasts

Usa directamente el CLI de Coast. No añadas wrappers.

## Orient Yourself

Empieza explorando el CLI y la documentación:

  coast                                             # see all available commands
  coast docs                                        # list all doc pages
  coast search-docs "your question"                 # semantic search

Cuando algo sobre el comportamiento de Coast no esté claro, lee la documentación antes de hacer suposiciones:

  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/PORTS.md
  coast docs --path coastfiles/README.md

## Quick Start

Enruta las solicitudes a uno de estos modos:

1. **Usar Coast** — ejecuta `coast lookup`, luego usa `coast exec`, `coast ps`
   o `coast logs` con la instancia correspondiente.
2. **Crear o asignar** — ejecuta `coast ls`, luego `coast run` para crear un nuevo
   Coast o `coast assign` para reapuntar uno existente.
3. **Abrir UI** — ejecuta `coast ui`.

## What Runs Where

El host y el Coast comparten el sistema de archivos. Usa `coast exec` solo para las cosas
que necesitan servicios en ejecución dentro del contenedor.

**Usa `coast exec` para:**
- Pruebas de integración, pruebas de API, cualquier cosa que necesite bases de datos o servicios
- Reinicios de servicios, operaciones de compose
- Comandos que se comuniquen con procesos solo disponibles en el contenedor

**Ejecuta en el host:**
- Linting (`eslint`, `rubocop`, `golangci-lint`)
- Verificación de tipos (`tsc --noEmit`, `go vet`)
- Formateo (`prettier`, `gofmt`)
- Operaciones de Git
- Playwright y pruebas de navegador
- Análisis estático, generación de código
- Instalación de paquetes (`npm install`, `pip install`)

## Create and Assign

Cuando `coast lookup` no devuelve ninguna coincidencia:

1. Ejecuta `coast ls` para ver los slots disponibles.
2. Prefiere `coast run <name> -w <worktree>` para crear y asignar en un solo paso.
3. Si todavía no existe ninguna compilación, ejecuta primero `coast build`.
4. Después de crear, vuelve a ejecutar `coast lookup` para confirmar.

Cuando quieras cambiar un Coast existente a un worktree diferente:

  coast assign <name> -w <worktree>

Eso también funciona para un Coast ya asignado o ya checked-out, pero consulta primero
con el usuario antes de reasignar un slot ocupado.

## Coastfile Setup

Si el proyecto necesita un Coastfile nuevo o modificado, primero lee la documentación:

  coast docs --path coastfiles/README.md

La documentación del Coastfile cubre la configuración de compose, puertos, volúmenes, secretos,
servicios compartidos, servicios sin contenedor auxiliar y herencia.

## Safety Rules

- Ejecuta `coast lookup` antes de actuar y de nuevo después de cualquier cambio de topología.
- Pregunta antes de `coast assign`, `coast unassign` o `coast checkout` si eso fuera a
  interrumpir un slot existente.
- Prefiere crear un nuevo Coast en lugar de reutilizar uno ya checked-out o ya asignado,
  a menos que el usuario quiera explícitamente que se reasigne el slot existente.
- Usa `coast docs` o `coast search-docs` antes de hacer suposiciones.
