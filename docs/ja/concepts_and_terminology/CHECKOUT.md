# チェックアウト

チェックアウトは、どの Coast インスタンスがあなたのプロジェクトの[カノニカルポート](PORTS.md)を所有するかを制御します。Coast をチェックアウトすると、`localhost:3000`、`localhost:5432`、および他のすべてのカノニカルポートが、そのインスタンスへ直接マップされます。

```bash
coast checkout dev-1
```

```text
Before checkout:
  localhost:3000  ──→  (nothing)
  localhost:5432  ──→  (nothing)

After checkout:
  localhost:3000  ──→  dev-1 web
  localhost:5432  ──→  dev-1 db
```

チェックアウトの切り替えは即時です — Coast は軽量な `socat` フォワーダーを停止して再生成します。コンテナは再起動されません。

```bash
coast checkout dev-2   # instant swap

# localhost:3000  ──→  dev-2 web
# localhost:5432  ──→  dev-2 db
```

## チェックアウトは必要ですか？

必ずしも必要ではありません。実行中のすべての Coast には常に固有の動的ポートがあり、何もチェックアウトしなくても、いつでもそれらのポート経由で任意の Coast にアクセスできます。

```bash
coast ports dev-1

# SERVICE    CANONICAL  DYNAMIC
# ★ web      3000       62217
#   db       5432       55681
```

チェックアウトしなくても、ブラウザで `localhost:62217` を開けば dev-1 の web サーバーに到達できます。これは多くのワークフローでまったく問題なく、`coast checkout` を一度も使わずに、好きなだけ Coast を実行できます。

## チェックアウトが有用なとき

動的ポートでは不十分で、カノニカルポートが必要になる状況があります。

- **カノニカルポートにハードコードされたクライアントアプリケーション。** Coast の外で動いているクライアント — ホスト上のフロントエンド開発サーバー、スマホ上のモバイルアプリ、デスクトップアプリ — が `localhost:3000` や `localhost:8080` を前提としている場合、あらゆる場所でポート番号を変更するのは非現実的です。Coast をチェックアウトすれば、設定を一切変更せずに実際のポートを使えます。

- **Webhook とコールバック URL。** Stripe、GitHub、OAuth プロバイダなどのサービスは、あなたが登録した URL（通常 `https://your-ngrok-tunnel.io` のように `localhost:3000` へ転送するもの）へコールバックを送ります。動的ポートに切り替えると、コールバックが届かなくなります。チェックアウトにより、テスト中の Coast に対してカノニカルポートが有効であることが保証されます。

- **データベースツール、デバッガ、IDE 連携。** 多くの GUI クライアント（pgAdmin、DataGrip、TablePlus）、デバッガ、IDE の実行構成は、特定のポートを持つ接続プロファイルを保存します。チェックアウトを使えば、保存済みプロファイルはそのままに、背後にある Coast を入れ替えるだけで済みます — コンテキストを切り替えるたびに、デバッガのアタッチ先やデータベース接続を再設定する必要はありません。

## チェックアウトの解除

別の Coast をチェックアウトせずにカノニカルポートを解放したい場合:

```bash
coast checkout --none
```

これ以降、どの Coast もカノニカルポートを所有しません。すべての Coast は動的ポート経由で引き続きアクセス可能です。

## 同時に 1 つだけ

チェックアウトできる Coast は同時に 1 つだけです。`dev-1` がチェックアウトされている状態で `coast checkout dev-2` を実行すると、カノニカルポートは即座に `dev-2` へ切り替わります。空白の時間はありません — 古いフォワーダーが停止され、新しいフォワーダーが同一操作内で生成されます。

```text
┌──────────────────────────────────────────────────┐
│  Your machine                                    │
│                                                  │
│  Canonical (checked-out Coast only):             │
│    localhost:3000 ──→ dev-2 web                  │
│    localhost:5432 ──→ dev-2 db                   │
│                                                  │
│  Dynamic (always available):                     │
│    localhost:62217 ──→ dev-1 web                 │
│    localhost:55681 ──→ dev-1 db                  │
│    localhost:63104 ──→ dev-2 web                 │
│    localhost:57220 ──→ dev-2 db                  │
└──────────────────────────────────────────────────┘
```

動的ポートはチェックアウトの影響を受けません。変わるのは、カノニカルポートがどこを指すかだけです。
