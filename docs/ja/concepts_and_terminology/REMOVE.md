# 削除

`coast rm` は Coast インスタンスを完全に削除します。実行中であればインスタンスを停止し、DinD コンテナを削除し、分離されたボリュームを削除し、ポートの割り当てを解除し、エージェントシェルを削除し、状態からインスタンスを削除します。

```bash
coast rm dev-1
```

日常的なワークフローのほとんどでは `coast rm` は必要ありません。単に別のコードを実行させたい、または正規のポートを所有させたいだけであれば、代わりに [Assign and
Unassign](ASSIGN.md) または [Checkout](CHECKOUT.md) を使用してください。`coast rm` は、Coast を停止したいとき、インスタンスごとのランタイム状態を回収したいとき、あるいは Coastfile やビルドを再構築したあとにインスタンスを最初から再作成したいときに使用してください。

## 何が起こるか

`coast rm` は 5 つのフェーズを実行します:

1. **検証と特定** — 状態内でインスタンスを検索します。状態
   レコードが失われていても、想定される名前のぶら下がったコンテナがまだ存在する場合、
   `coast rm` はそれもクリーンアップします。
2. **必要なら停止** — インスタンスが `Running` または `CheckedOut` の場合、Coast
   はまず内部の compose スタックを停止し、その後 DinD コンテナを停止します。
3. **ランタイム成果物の削除** — Coast コンテナを削除し、そのインスタンスの
   分離されたボリュームを削除します。
4. **ホスト状態のクリーンアップ** — 残っているポートフォワーダーを終了し、ポートの割り当てを解除し、エージェントシェルを削除し、状態
   データベースからインスタンスレコードを削除します。
5. **共有データの保持** — 共有サービスボリュームと共有サービスデータは
   そのまま残されます。

## CLI の使い方

```text
coast rm <name>
coast rm --all
```

| Flag | Description |
|------|-------------|
| `<name>` | 名前で 1 つのインスタンスを削除 |
| `--all` | 現在のプロジェクトのすべてのインスタンスを削除 |

`coast rm --all` は現在のプロジェクトを解決し、そのインスタンスを一覧表示して、
それらを 1 つずつ削除します。インスタンスが存在しない場合は、正常に終了します。

## 共有サービスとビルド

- `coast rm` は共有サービスデータを削除しません。
- 共有
  サービスとそのデータも削除したい場合は、`coast shared-services rm <service>` を使用してください。
- インスタンスを停止したあとでビルド成果物を削除したい場合は、
  `coast rm-build` を使用してください。

## 使用するタイミング

- Coastfile を再構築したあと、または新しいビルドを作成したあとで、新しい
  インスタンスが必要なとき
- Coast を停止し、インスタンスごとのコンテナおよびボリュームの
  状態を解放したいとき
- インスタンスが行き詰まっており、その場でデバッグするよりも最初からやり直すほうが簡単なとき
