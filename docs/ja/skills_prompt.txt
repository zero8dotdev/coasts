# Coast Runtime

このプロジェクトでは、分離された開発環境のために Coasts（containerized host）を使用します。
あなたのコード編集は、実行中の Coast 内に自動的に反映されます — ファイルシステムは
ホストとコンテナの間で共有されています。

## Orientation

ランタイムコマンドを実行する前に、現在の作業ディレクトリに一致する Coast インスタンスが
どれかを確認してください:

```
coast lookup
```

これにより、インスタンス名、ポート、URL、およびコマンド例が表示されます。以降のすべてのコマンドでは、
出力にあるインスタンス名を使用してください。

Coasts の仕組みについてより深い文脈が必要な場合は、以下のドキュメントを読んでください:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

Coast 内でコマンドを実行するには `coast exec` を使用します。シェルはワークスペース
ルート（Coastfile がある場所）で開始されます。まず対象ディレクトリに cd してください:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

例:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

サービスの状態を確認します:

```
coast ps <instance>
```

サービスログを読みます:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

エラーや見慣れない挙動に遭遇した場合は、Coast ドキュメントを検索してください:

```
coast search-docs "error message or description"
```

これはセマンティック検索を使用します — 問題を自然言語で説明すると、
関連するドキュメントを見つけます。

## Worktree Awareness

worktree で作業を始めるときは — 自分で作成した場合でも、Codex、Conductor、または
T3 Code のようなツールが作成した場合でも — すでに Coast インスタンスが割り当て
られているか確認してください:

```
coast lookup
```

`coast lookup` がインスタンスを見つけた場合は、すべてのランタイムコマンドでそれを使用してください。

インスタンスが返されない場合は、現在実行中のものを確認してください:

```
coast ls
```

その後、どの選択肢を希望するかをユーザーに尋ねてください:

**Option 1 — 新しい Coast を作成してこの worktree を割り当てる:**
```
coast run <new-name>
coast assign <new-name> -w <worktree>
```

**Option 2 — 既存の Coast をこの worktree に再割り当てする:**
```
coast assign <existing-name> -w <worktree>
```

**Option 3 — Coast を完全に使わずに進める:**
ランタイム環境なしで続行します。ファイルは編集できますが、コンテナ内でテスト、
ビルド、またはサービスを実行することはできません。

`<worktree>` の値はブランチ名（`git branch --show-current` を実行）または
`coast ls` に表示される worktree 識別子です。作成または再割り当ての前には、
必ずユーザーに確認してください — 自動では行わないでください。

## Rules

- セッション内で最初のランタイムコマンドを実行する前に、必ず `coast lookup` を実行してください。
- ホスト上で直接サービスを実行しないでください。すべてのランタイムタスクには `coast exec` を使用してください。
- ホスト上でのファイル編集は即座に Coast 内に反映されます。編集後にファイルをコピーしたり
  再ビルドしたりする必要はありません。
- `coast lookup` がインスタンスを返さない場合、Coast が実行されていない可能性があります。
  これをユーザーと解決するために、上記の worktree awareness フローに従ってください。
