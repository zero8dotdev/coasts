# Coast Runtime

このプロジェクトでは、サービス、テスト、その他のランタイムコマンドを実行するために Coasts（コンテナ化されたランタイム）を使用します。ファイルシステムはホストとコンテナの間で共有されるため、どちらか一方で行ったファイル編集も即座に両方から見えるようになります。

## Discovery

セッション内で最初のランタイムコマンドを実行する前に、次を実行してください:

  coast lookup

これにより、インスタンス名、ポート、およびコマンド例が表示されます。以降のすべてのコマンドでは、出力にあるインスタンス名を使用してください。

## What runs where

ファイルシステムは共有されているため、`coast exec` はコンテナランタイムが必要なもの（データベース、サービス、統合テスト）にだけ使用してください。それ以外はすべてホスト上で直接実行します。

`coast exec` を使うもの:
- 実行中のサービスが必要なテスト（サービスや db と統合されたユニットテスト、統合テスト、API テスト）
- サービスの再起動や compose 操作
- データベース、キャッシュ、その他のコンテナサービスとやり取りするもの

ホスト上で直接実行するもの:
- リント、型チェック、フォーマット
- Git 操作
- Playwright とブラウザテスト
- ホスト側依存関係のインストール (`npm install`, `pip install`)
- ファイル検索、コード生成、静的解析

例:

  coast exec <instance> -- sh -c "cd <dir> && npm test"    # DB が必要
  npm run lint                                              # ホストで問題なし
  npx playwright test                                       # ホストで問題なし

## Runtime feedback

  coast ps <instance>
  coast logs <instance> --service <service>
  coast logs <instance> --service <service> --tail 50

## Creating and assigning Coasts

`coast lookup` が一致を返さない場合は、`coast ls` を実行して何が存在するか確認してください。

このプロジェクト用の未割り当ての Coast がすでに実行中であれば、新しく作成するよりも、まずそれに自分の worktree を割り当てることを優先してください:

  coast assign <existing> -w <worktree>

すでに使用中の Coast も `coast assign` で再割り当てできますが、現在のスロットを妨げることになるため、必ず先にユーザーへ確認してください。

実行中の Coast がない場合は、作成前にユーザーへ確認してください — Coasts はメモリ消費が大きいことがあります:

  coast run <name> -w <worktree>

インスタンスを作成する前に、プロジェクトはビルドされている必要があります。ビルドが存在しないために `coast run` が失敗した場合は、まず `coast build` を実行してください。

## Coastfile setup

プロジェクトにまだ Coastfile がない場合、または Coastfile を変更する必要がある場合は、まず Coastfile のドキュメントを読んでください:

  coast docs --path coastfiles/README.md

## When confused

Coast の挙動を推測する前に、ドキュメントを確認してください:

  coast docs                                     # すべてのドキュメントページを一覧表示
  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast search-docs "your question here"         # セマンティック検索

## Rules

- セッション内で最初のランタイムコマンドを実行する前に、必ず `coast lookup` を実行してください。
- `coast exec` はコンテナランタイムが必要なものにだけ使用してください。
- リント、型チェック、フォーマット、および git はホスト上で直接実行してください。
- Coast の挙動を推測する前に `coast docs` または `coast search-docs` を使ってください。
- プロジェクトが Coast を前提としている場合、ホスト上でサービスを直接実行しないでください。

# Coasts

Coast CLI を直接使用してください。ラッパーは追加しないでください。

## Orient Yourself

まず CLI とドキュメントを確認してください:

  coast                                             # 使用可能なすべてのコマンドを表示
  coast docs                                        # すべてのドキュメントページを一覧表示
  coast search-docs "your question"                 # セマンティック検索

Coast の挙動について不明な点があれば、推測する前にドキュメントを読んでください:

  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/PORTS.md
  coast docs --path coastfiles/README.md

## Quick Start

リクエストを次のいずれかのモードに振り分けてください:

1. **Use Coast** — `coast lookup` を実行し、その後、一致したインスタンスに対して `coast exec`、`coast ps`、または `coast logs` を使用します。
2. **Create or Assign** — `coast ls` を実行し、その後 `coast run` で新しい Coast を作成するか、`coast assign` で既存のものの向き先を変更します。
3. **Open UI** — `coast ui` を実行します。

## What Runs Where

ホストと Coast はファイルシステムを共有しています。`coast exec` は、コンテナ内で実行中のサービスが必要なものにだけ使用してください。

**`coast exec` を使うもの:**
- 統合テスト、API テスト、その他データベースやサービスが必要なもの
- サービスの再起動、compose 操作
- コンテナ内にしかないプロセスとやり取りするコマンド

**ホスト上で実行するもの:**
- リント (`eslint`, `rubocop`, `golangci-lint`)
- 型チェック (`tsc --noEmit`, `go vet`)
- フォーマット (`prettier`, `gofmt`)
- Git 操作
- Playwright とブラウザテスト
- 静的解析、コード生成
- パッケージのインストール (`npm install`, `pip install`)

## Create and Assign

`coast lookup` が一致を返さない場合:

1. `coast ls` を実行して利用可能なスロットを確認します。
2. 作成と割り当てを 1 ステップで行うために、`coast run <name> -w <worktree>` を優先してください。
3. まだビルドが存在しない場合は、まず `coast build` を実行してください。
4. 作成後、確認のために `coast lookup` を再実行してください。

既存の Coast を別の worktree に切り替えたい場合:

  coast assign <name> -w <worktree>

これは、すでに割り当て済みまたはチェックアウト済みの Coast に対しても機能しますが、使用中のスロットを再割り当てする前には必ずユーザーへ確認してください。

## Coastfile Setup

プロジェクトで新規または変更された Coastfile が必要な場合は、まずドキュメントを読んでください:

  coast docs --path coastfiles/README.md

Coastfile のドキュメントでは、compose 設定、ポート、ボリューム、シークレット、共有サービス、bare services、および継承を扱っています。

## Safety Rules

- 何か行動を起こす前に `coast lookup` を実行し、トポロジー変更後にも再度実行してください。
- 既存のスロットを妨げる可能性がある場合、`coast assign`、`coast unassign`、または `coast checkout` の前に確認してください。
- ユーザーが既存スロットの再割り当てを明示的に望んでいない限り、チェックアウト済みまたはすでに割り当て済みのものを再利用するより、新しい Coast を作成することを優先してください。
- 推測する前に `coast docs` または `coast search-docs` を使ってください。
