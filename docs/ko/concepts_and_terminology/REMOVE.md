# 제거

`coast rm`은 Coast 인스턴스를 완전히 제거합니다. 인스턴스가 실행 중이면 중지하고,
DinD 컨테이너를 제거하며, 격리된 볼륨을 삭제하고, 포트를 할당 해제하며, 에이전트 셸을 제거하고, 상태에서 인스턴스를 삭제합니다.

```bash
coast rm dev-1
```

대부분의 일상적인 워크플로에서는 `coast rm`이 필요하지 않습니다. Coast가 다른 코드를
실행하게 하거나 정식 포트를 소유하게 하려는 것뿐이라면, 대신 [할당 및
할당 해제](ASSIGN.md) 또는 [체크아웃](CHECKOUT.md)을 사용하세요. Coasts를 내리고,
인스턴스별 런타임 상태를 회수하거나, Coastfile 또는 빌드를 다시 만든 뒤 인스턴스를 처음부터
다시 생성하고 싶을 때 `coast rm`을 사용하세요.

## 어떤 일이 일어나는가

`coast rm`은 다섯 단계로 실행됩니다:

1. **검증 및 위치 확인** — 상태에서 인스턴스를 조회합니다. 상태 레코드는
   사라졌지만 예상된 이름의 매달린 컨테이너가 여전히 존재한다면,
   `coast rm`은 그것도 정리합니다.
2. **필요 시 중지** — 인스턴스가 `Running` 또는 `CheckedOut` 상태이면, Coast는
   먼저 내부 compose 스택을 내리고 DinD 컨테이너를 중지합니다.
3. **런타임 아티팩트 제거** — Coast 컨테이너를 제거하고 해당 인스턴스의
   격리된 볼륨을 삭제합니다.
4. **호스트 상태 정리** — 남아 있는 포트 포워더를 종료하고, 포트를 할당 해제하며,
   에이전트 셸을 제거하고, 상태 데이터베이스에서 인스턴스 레코드를 삭제합니다.
5. **공유 데이터 보존** — 공유 서비스 볼륨과 공유 서비스 데이터는
   그대로 둡니다.

## CLI 사용법

```text
coast rm <name>
coast rm --all
```

| Flag | Description |
|------|-------------|
| `<name>` | 이름으로 하나의 인스턴스 제거 |
| `--all` | 현재 프로젝트의 모든 인스턴스 제거 |

`coast rm --all`은 현재 프로젝트를 확인하고, 해당 인스턴스 목록을 나열한 뒤,
하나씩 제거합니다. 인스턴스가 없으면 문제없이 종료합니다.

## 공유 서비스와 빌드

- `coast rm`은 공유 서비스 데이터를 삭제하지 **않습니다**.
- 공유 서비스와 그 데이터도 함께 제거하려면 `coast shared-services rm <service>`를 사용하세요.
- 인스턴스를 내린 후 빌드 아티팩트를 제거하려면 `coast rm-build`를 사용하세요.

## 언제 사용해야 하는가

- Coastfile을 다시 만들거나 새 빌드를 만든 뒤 새 인스턴스를 원할 때
- Coasts를 내리고 인스턴스별 컨테이너 및 볼륨 상태를 해제하고 싶을 때
- 인스턴스가 꼬여서 제자리에서 디버깅하는 것보다 새로 시작하는 편이 더 쉬울 때
