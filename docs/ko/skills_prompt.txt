# Coast Runtime

이 프로젝트는 서비스, 테스트 및 기타 런타임 명령을 실행하기 위해 Coasts — 컨테이너화된 런타임 — 를 사용합니다. 파일 시스템은 호스트와 컨테이너 사이에 공유되므로, 어느 쪽에서든 파일을 편집하면 양쪽에 즉시 반영됩니다.

## Discovery

세션에서 첫 번째 런타임 명령을 실행하기 전에 다음을 실행하세요:

  coast lookup

이 명령은 인스턴스 이름, 포트, 예시 명령을 출력합니다. 이후의 모든 명령에는 출력에 나온 인스턴스 이름을 사용하세요.

## What runs where

파일 시스템은 공유되므로, 컨테이너 런타임(데이터베이스, 서비스, 통합 테스트)이 필요한 작업에만 `coast exec`를 사용하세요. 그 외의 모든 것은 호스트에서 직접 실행합니다.

다음에는 `coast exec`를 사용하세요:
- 실행 중인 서비스가 필요한 테스트(서비스나 DB와 통합된 단위 테스트, 통합 테스트, API 테스트)
- 서비스 재시작 또는 compose 작업
- 데이터베이스, 캐시 또는 기타 컨테이너 서비스와 통신하는 모든 작업

다음은 호스트에서 직접 실행하세요:
- 린팅, 타입체킹, 포매팅
- Git 작업
- Playwright 및 브라우저 테스트
- 호스트 측 의존성 설치(npm install, pip install)
- 파일 검색, 코드 생성, 정적 분석

예시:

  coast exec <instance> -- sh -c "cd <dir> && npm test"    # DB 필요
  npm run lint                                              # 호스트에서 충분
  npx playwright test                                       # 호스트에서 충분

## Runtime feedback

  coast ps <instance>
  coast logs <instance> --service <service>
  coast logs <instance> --service <service> --tail 50

## Creating and assigning Coasts

`coast lookup`이 일치하는 항목을 반환하지 않으면, 무엇이 있는지 보기 위해 `coast ls`를 실행하세요.

이 프로젝트에 대해 아직 할당되지 않은 Coast가 이미 실행 중이라면, 새로 만드는 것보다 해당 Coast에 워크트리를 할당하는 것을 우선하세요:

  coast assign <existing> -w <worktree>

이미 사용 중인 Coast도 `coast assign`으로 재할당할 수 있지만, 현재 슬롯이 중단되므로 먼저 사용자에게 확인하세요.

실행 중인 Coast가 없다면, 새로 만들기 전에 사용자에게 물어보세요 — Coasts는 메모리를 많이 사용할 수 있습니다:

  coast run <name> -w <worktree>

인스턴스를 생성하려면 먼저 프로젝트가 빌드되어 있어야 합니다. 빌드가 없어서 `coast run`이 실패하면 먼저 `coast build`를 실행하세요.

## Coastfile setup

프로젝트에 아직 Coastfile이 없거나 Coastfile을 수정해야 한다면, 먼저 Coastfile 문서를 읽으세요:

  coast docs --path coastfiles/README.md

## When confused

Coast 동작을 추측하기 전에 문서를 살펴보세요:

  coast docs                                     # 모든 문서 페이지 나열
  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast search-docs "your question here"         # 시맨틱 검색

## Rules

- 세션에서 첫 번째 런타임 명령 전에 항상 `coast lookup`을 실행하세요.
- 컨테이너 런타임이 필요한 작업에만 `coast exec`를 사용하세요.
- 린팅, 타입체킹, 포매팅, git은 호스트에서 직접 실행하세요.
- Coast 동작을 추측하기 전에 `coast docs` 또는 `coast search-docs`를 사용하세요.
- 프로젝트가 Coast를 기대하는 경우 호스트에서 서비스를 직접 실행하지 마세요.

# Coasts

Coast CLI를 직접 사용하세요. 래퍼를 추가하지 마세요.

## Orient Yourself

먼저 CLI와 문서를 살펴보세요:

  coast                                             # 사용 가능한 모든 명령 보기
  coast docs                                        # 모든 문서 페이지 나열
  coast search-docs "your question"                 # 시맨틱 검색

Coast 동작과 관련해 불분명한 점이 있으면, 추측하기 전에 문서를 읽으세요:

  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/PORTS.md
  coast docs --path coastfiles/README.md

## Quick Start

요청을 다음 모드 중 하나로 분기하세요:

1. **Use Coast** — `coast lookup`을 실행한 다음, 일치하는 인스턴스에 대해 `coast exec`, `coast ps`, 또는 `coast logs`를 사용합니다.
2. **Create or Assign** — `coast ls`를 실행한 다음, `coast run`으로 새 Coast를 만들거나 `coast assign`으로 기존 Coast가 가리키는 대상을 변경합니다.
3. **Open UI** — `coast ui`를 실행합니다.

## What Runs Where

호스트와 Coast는 파일 시스템을 공유합니다. 컨테이너 내부에서 실행 중인 서비스가 필요한 작업에만 `coast exec`를 사용하세요.

**다음에는 `coast exec`를 사용하세요:**
- 통합 테스트, API 테스트, 데이터베이스나 서비스가 필요한 모든 작업
- 서비스 재시작, compose 작업
- 컨테이너에서만 실행되는 프로세스와 통신하는 명령

**다음은 호스트에서 실행하세요:**
- 린팅 (`eslint`, `rubocop`, `golangci-lint`)
- 타입체킹 (`tsc --noEmit`, `go vet`)
- 포매팅 (`prettier`, `gofmt`)
- Git 작업
- Playwright 및 브라우저 테스트
- 정적 분석, 코드 생성
- 패키지 설치 (`npm install`, `pip install`)

## Create and Assign

`coast lookup`이 일치하는 항목을 반환하지 않을 때:

1. 사용 가능한 슬롯을 보기 위해 `coast ls`를 실행하세요.
2. 한 번에 생성하고 할당하려면 `coast run <name> -w <worktree>`를 우선 사용하세요.
3. 아직 빌드가 없다면 먼저 `coast build`를 실행하세요.
4. 생성 후 확인을 위해 `coast lookup`을 다시 실행하세요.

기존 Coast를 다른 워크트리로 전환하려는 경우:

  coast assign <name> -w <worktree>

이 방법은 이미 할당되었거나 체크아웃된 Coast에도 동작하지만, 사용 중인 슬롯을 재할당하기 전에는 먼저 사용자에게 물어보세요.

## Coastfile Setup

프로젝트에 새 Coastfile이 필요하거나 기존 Coastfile을 수정해야 한다면, 먼저 문서를 읽으세요:

  coast docs --path coastfiles/README.md

Coastfile 문서에는 compose 설정, 포트, 볼륨, 시크릿, 공유 서비스, bare 서비스, 상속이 포함되어 있습니다.

## Safety Rules

- 조치를 취하기 전에 `coast lookup`을 실행하고, 토폴로지가 변경된 후에도 다시 실행하세요.
- `coast assign`, `coast unassign`, 또는 `coast checkout`이 기존 슬롯을 방해하게 된다면 먼저 물어보세요.
- 사용자가 기존 슬롯의 재할당을 명시적으로 원하지 않는 한, 체크아웃되었거나 이미 할당된 Coast를 재사용하는 것보다 새 Coast를 만드는 것을 우선하세요.
- 추측하기 전에 `coast docs` 또는 `coast search-docs`를 사용하세요.
