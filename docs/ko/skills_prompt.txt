# Coast Runtime

이 프로젝트는 격리된 개발 환경을 위해 Coast(컨테이너화된 호스트)를 사용합니다.
작성한 코드 수정 사항은 실행 중인 Coast 내부에 자동으로 표시됩니다 — 파일 시스템은
호스트와 컨테이너 간에 공유됩니다.

## Orientation

런타임 명령을 실행하기 전에, 현재 작업 디렉터리와 일치하는 Coast 인스턴스를
확인하세요:

```
coast lookup
```

이 명령은 인스턴스 이름, 포트, URL 및 예시 명령을 출력합니다. 이후의 모든 명령에는
출력에 있는 인스턴스 이름을 사용하세요.

Coast가 어떻게 동작하는지에 대한 더 깊은 맥락이 필요하다면, 다음 문서를 읽으세요:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

Coast 내부에서 명령을 실행하려면 `coast exec`를 사용하세요. 셸은 작업 공간
루트(Coastfile이 있는 위치)에서 시작됩니다. 먼저 대상 디렉터리로 cd 하세요:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

예시:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

서비스 상태 확인:

```
coast ps <instance>
```

서비스 로그 읽기:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

오류나 익숙하지 않은 동작이 발생하면, Coast 문서를 검색하세요:

```
coast search-docs "error message or description"
```

이것은 시맨틱 검색을 사용합니다 — 문제를 자연어로 설명하면 관련 문서를
찾아줍니다.

## Worktree Awareness

워크트리에서 작업을 시작할 때 — 직접 생성했든, Codex, Conductor 또는
T3 Code 같은 도구가 대신 생성했든 — 이미 Coast 인스턴스가 할당되어 있는지
확인하세요:

```
coast lookup
```

`coast lookup`이 인스턴스를 찾으면, 모든 런타임 명령에 그것을 사용하세요.

인스턴스를 반환하지 않으면, 현재 실행 중인 항목을 확인하세요:

```
coast ls
```

그런 다음 사용자가 어떤 옵션을 선호하는지 물어보세요:

**옵션 1 — 새 Coast를 생성하고 이 워크트리를 할당:**
```
coast run <new-name>
coast assign <new-name> -w <worktree>
```

**옵션 2 — 기존 Coast를 이 워크트리에 재할당:**
```
coast assign <existing-name> -w <worktree>
```

**옵션 3 — Coast를 완전히 건너뛰기:**
런타임 환경 없이 계속 진행합니다. 파일은 편집할 수 있지만, 컨테이너 내부에서
테스트, 빌드 또는 서비스를 실행할 수는 없습니다.

`<worktree>` 값은 브랜치 이름(`git branch --show-current` 실행) 또는
`coast ls`에 표시되는 워크트리 식별자입니다. 생성하거나 재할당하기 전에 항상
사용자에게 물어보세요 — 자동으로 수행하지 마세요.

## Rules

- 세션에서 첫 번째 런타임 명령을 실행하기 전에 항상 `coast lookup`을 실행하세요.
- 호스트에서 직접 서비스를 실행하지 마세요. 모든 런타임 작업에는 `coast exec`를 사용하세요.
- 호스트에서의 파일 편집은 Coast 내부에 즉시 표시됩니다. 편집 후 파일을 복사하거나
  다시 빌드할 필요가 없습니다.
- `coast lookup`이 인스턴스를 반환하지 않으면, Coast가 실행 중이 아닐 수 있습니다. 이를
  사용자와 함께 해결하려면 위의 워크트리 인식 흐름을 따르세요.

=== 스킬 텍스트 끝 ===

3. 파일을 작성한 후, `ls Coastfile`을 실행하여 이 프로젝트에 Coastfile이 존재하는지 확인합니다.
4. 이미 실행 중인 Coast가 있다면, `coast lookup`을 실행하여 스킬이 작동하는지 확인합니다. 실행 중인 Coast가 없다면, 사용자가 `coast run dev-1`로 시작할 수 있다고 알려주세요.
