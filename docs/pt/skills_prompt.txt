# Coast Runtime

Este projeto usa Coasts — runtimes conteinerizados para executar serviços, testes
e outros comandos de runtime. O sistema de arquivos é compartilhado entre o host e o
contêiner, então edições de arquivos em qualquer lado ficam visíveis para ambos imediatamente.

## Discovery

Antes do primeiro comando de runtime em uma sessão, execute:

  coast lookup

Isso imprime o nome da instância, portas e comandos de exemplo. Use o nome da instância
da saída para todos os comandos subsequentes.

## What runs where

O sistema de arquivos é compartilhado, então use `coast exec` apenas para coisas que precisam do
runtime do contêiner (bancos de dados, serviços, testes de integração). Todo o resto
é executado diretamente no host.

Use `coast exec` para:
- Testes que precisam de serviços em execução (testes unitários integrados com serviços ou bancos de dados, testes de integração, testes de API)
- Reinicializações de serviços ou operações de compose
- Qualquer coisa que se comunique com bancos de dados, caches ou outros serviços em contêineres

Execute diretamente no host:
- Linting, verificação de tipos, formatação
- Operações de git
- Testes com Playwright e testes de navegador
- Instalação de dependências do lado do host (npm install, pip install)
- Busca de arquivos, geração de código, análise estática

Exemplo:

  coast exec <instance> -- sh -c "cd <dir> && npm test"    # needs DB
  npm run lint                                              # host is fine
  npx playwright test                                       # host is fine

## Runtime feedback

  coast ps <instance>
  coast logs <instance> --service <service>
  coast logs <instance> --service <service> --tail 50

## Creating and assigning Coasts

Se `coast lookup` não retornar correspondência, execute `coast ls` para ver o que existe.

Se um Coast não atribuído já estiver em execução para este projeto, prefira atribuir
sua worktree a ele em vez de criar um novo:

  coast assign <existing> -w <worktree>

Um Coast já ocupado também pode ser reatribuído com `coast assign`, mas confirme
com o usuário primeiro, porque isso vai interromper o slot atual.

Se nenhum Coast estiver em execução, pergunte ao usuário antes de criar um — Coasts podem consumir
muita memória:

  coast run <name> -w <worktree>

Um projeto precisa ser compilado antes que instâncias possam ser criadas. Se `coast run` falhar
porque não existe nenhum build, execute `coast build` primeiro.

## Coastfile setup

Se o projeto ainda não tiver um Coastfile, ou se você precisar modificar o
Coastfile, leia primeiro a documentação do Coastfile:

  coast docs --path coastfiles/README.md

## Rules

- Sempre execute `coast lookup` antes do seu primeiro comando de runtime em uma sessão.
- Use `coast exec` apenas para coisas que precisam do runtime do contêiner.
- Execute linting, verificação de tipos, formatação e git diretamente no host.
- Use `coast docs` ou `coast search-docs` antes de supor algo sobre o comportamento do Coast.
- Não execute serviços diretamente no host quando o projeto espera Coast.

# Coasts

Use a CLI do Coast diretamente. Não adicione wrappers.

## Orient Yourself

Comece explorando a CLI e a documentação:

  coast                                             # see all available commands
  coast docs                                        # list all doc pages
  coast search-docs "your question"                 # semantic search

Quando qualquer aspecto do comportamento do Coast estiver incerto, leia a documentação antes de supor:

  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/PORTS.md
  coast docs --path coastfiles/README.md

## Quick Start

Encaminhe as solicitações para um destes modos:

1. **Use Coast** — execute `coast lookup`, depois use `coast exec`, `coast ps`,
   ou `coast logs` com a instância correspondente.
2. **Create or Assign** — execute `coast ls`, depois `coast run` para criar um novo
   Coast ou `coast assign` para redirecionar um existente.
3. **Open UI** — execute `coast ui`.

## What Runs Where

O host e o Coast compartilham o sistema de arquivos. Use `coast exec` apenas para coisas
que precisam de serviços em execução dentro do contêiner.

**Use `coast exec` for:**
- Testes de integração, testes de API, qualquer coisa que precise de bancos de dados ou serviços
- Reinicializações de serviços, operações de compose
- Comandos que se comunicam com processos disponíveis apenas no contêiner

**Run on the host:**
- Linting (`eslint`, `rubocop`, `golangci-lint`)
- Verificação de tipos (`tsc --noEmit`, `go vet`)
- Formatação (`prettier`, `gofmt`)
- Operações de git
- Testes com Playwright e testes de navegador
- Análise estática, geração de código
- Instalação de pacotes (`npm install`, `pip install`)

## Create and Assign

Quando `coast lookup` não retornar correspondência:

1. Execute `coast ls` para ver os slots disponíveis.
2. Prefira `coast run <name> -w <worktree>` para criar e atribuir em uma única etapa.
3. Se ainda não existir nenhum build, execute `coast build` primeiro.
4. Após criar, execute `coast lookup` novamente para confirmar.

Quando você quiser trocar um Coast existente para uma worktree diferente:

  coast assign <name> -w <worktree>

Isso também funciona para um Coast já atribuído ou com checkout feito, mas pergunte ao usuário
primeiro antes de reatribuir um slot ocupado.

## Coastfile Setup

Se o projeto precisar de um Coastfile novo ou modificado, leia a documentação primeiro:

  coast docs --path coastfiles/README.md

A documentação do Coastfile cobre configuração de compose, portas, volumes, segredos, serviços
compartilhados, serviços bare e herança.

## Safety Rules

- Execute `coast lookup` antes de agir e novamente após qualquer mudança de topologia.
- Pergunte antes de `coast assign`, `coast unassign` ou `coast checkout` se isso puder
  interromper um slot existente.
- Prefira criar um novo Coast em vez de reutilizar um que já tenha checkout feito ou já esteja atribuído,
  a menos que o usuário queira explicitamente que o slot existente seja reatribuído.
- Use `coast docs` ou `coast search-docs` antes de supor algo.
