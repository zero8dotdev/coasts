# Coast Runtime

Este projeto usa Coasts (host conteinerizado) para ambientes de desenvolvimento isolados.
Suas edições de código ficam automaticamente visíveis dentro do Coast em execução — o sistema de arquivos
é compartilhado entre o host e o contêiner.

## Orientation

Antes de executar quaisquer comandos de runtime, descubra qual instância do Coast corresponde ao seu
diretório de trabalho atual:

```
coast lookup
```

Isso imprime o nome da instância, portas, URLs e comandos de exemplo. Use o nome da instância
da saída para todos os comandos subsequentes.

Se você precisar de um contexto mais profundo sobre como os Coasts funcionam, leia estes documentos:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

Use `coast exec` para executar comandos dentro do Coast. O shell inicia na raiz do workspace
(onde está o Coastfile). Primeiro, faça cd para o diretório de destino:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

Exemplos:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

Verifique o status do serviço:

```
coast ps <instance>
```

Leia os logs do serviço:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

Se você encontrar erros ou comportamento incomum, pesquise na documentação do Coast:

```
coast search-docs "mensagem de erro ou descrição"
```

Isto usa busca semântica — descreva o problema em linguagem natural e ela
encontrará a documentação relevante.

## Worktree Awareness

Quando você começar a trabalhar em uma worktree — quer ela tenha sido criada por você ou por uma ferramenta como
Codex, Conductor ou T3 Code — verifique se uma instância do Coast já está
atribuída:

```
coast lookup
```

Se `coast lookup` encontrar uma instância, use-a para todos os comandos de runtime.

Se não retornar nenhuma instância, verifique o que está em execução no momento:

```
coast ls
```

Então pergunte ao usuário qual opção ele prefere:

**Option 1 — Create a new Coast and assign this worktree:**
```
coast run <new-name>
coast assign <new-name> -w <worktree>
```

**Option 2 — Reassign an existing Coast to this worktree:**
```
coast assign <existing-name> -w <worktree>
```

**Option 3 — Skip Coast entirely:**
Continue sem um ambiente de runtime. Você pode editar arquivos, mas não pode executar testes,
builds ou serviços dentro de um contêiner.

O valor de `<worktree>` é o nome da branch (execute `git branch --show-current`) ou
o identificador da worktree mostrado em `coast ls`. Sempre pergunte ao usuário antes de criar
ou reatribuir — não faça isso automaticamente.

## Rules

- Sempre execute `coast lookup` antes do seu primeiro comando de runtime em uma sessão.
- Não execute serviços diretamente no host. Use `coast exec` para todas as tarefas de runtime.
- As edições de arquivos no host são visíveis instantaneamente dentro do Coast. Você não precisa
  copiar arquivos nem reconstruir após editar.
- Se `coast lookup` não retornar nenhuma instância, o Coast pode não estar em execução. Siga o
  fluxo de reconhecimento de worktree acima para resolver isso com o usuário.
