# Документация Coasts

## Установка

- `curl -fsSL https://coasts.dev/install | sh`
- `coast daemon install`

*Если вы решите не запускать `coast daemon install`, вы несёте ответственность за ручной запуск демона с помощью `coast daemon start` каждый раз.*

## Что такое Coasts?

Coast (**контейнеризованный хост**) — это локальная среда выполнения для разработки. Coasts позволяют запускать несколько изолированных окружений для одного и того же проекта на одной машине.

Coasts особенно полезны для сложных стеков `docker-compose` со множеством взаимозависимых сервисов, но они столь же эффективны и для не контейнеризованных локальных dev-настроек. Coasts поддерживают широкий спектр [шаблонов конфигурации среды выполнения](concepts_and_terminology/RUNTIMES_AND_SERVICES.md), чтобы вы могли сформировать идеальную среду для нескольких агентов, работающих параллельно.

Coasts созданы для локальной разработки, а не как хостинговый облачный сервис. Ваши окружения работают локально на вашей машине.

Проект Coasts — это бесплатное локальное ПО под лицензией MIT, не привязанное ни к провайдеру агентов, ни к harness’у, без каких-либо AI-допродаж.

Coasts работают с любым агентным workflow для кодинга, который использует worktrees. Никакой специальной настройки на стороне harness не требуется.

## Почему Coasts для Worktrees

Git worktrees отлично подходят для изоляции изменений кода, но сами по себе они не решают изоляцию среды выполнения.

Когда вы запускаете несколько worktrees параллельно, вы быстро сталкиваетесь с проблемами удобства:

- [Конфликты портов](concepts_and_terminology/PORTS.md) между сервисами, которые ожидают одни и те же хост-порты.
- Настройка базы данных и [томов](concepts_and_terminology/VOLUMES.md) для каждого worktree, которой утомительно управлять.
- Окружения интеграционных тестов, которым требуется кастомная «проводка» среды выполнения для каждого worktree.
- Ад при переключении worktrees и пересборке контекста среды выполнения каждый раз. См. [Assign and Unassign](concepts_and_terminology/ASSIGN.md).

Если Git — это контроль версий для вашего кода, то Coasts — это как Git для сред выполнения ваших worktrees.

Каждое окружение получает свои собственные порты, поэтому вы можете параллельно инспектировать среду выполнения любого worktree. Когда вы [делаете checkout](concepts_and_terminology/CHECKOUT.md) среды выполнения worktree, Coasts переназначают эту среду выполнения на канонические порты вашего проекта.

Coasts абстрагируют конфигурацию среды выполнения в простой модульный слой поверх worktrees, чтобы каждый worktree мог запускаться с нужной ему изоляцией без ручного сопровождения сложной настройки для каждого worktree.

- macOS
- Docker Desktop
- Проект, использующий Git
- Node.js
- `socat` *(устанавливается с `curl -fsSL https://coasts.dev/install | sh` как зависимость Homebrew `depends_on`)*

## Контейнеризировать агентов?

Вы можете контейнеризировать агента с помощью Coast. Поначалу это может показаться отличной идеей, но во многих случаях вам на самом деле не нужно запускать вашего coding-агента внутри контейнера.

Поскольку Coasts разделяют [файловую систему](concepts_and_terminology/FILESYSTEM.md) с вашей хост-машиной через общий volume mount, самый простой и надёжный workflow — запускать агента на хосте и поручать ему выполнять ресурсоёмкие задачи среды выполнения (например, интеграционные тесты) внутри инстанса Coast с помощью [`coast exec`](concepts_and_terminology/EXEC_AND_DOCKER.md).

Однако если вы всё же хотите запускать агента в контейнере, Coasts безусловно поддерживают это через [Agent Shells](concepts_and_terminology/AGENT_SHELLS.md). Вы можете собрать невероятно сложный риг для этой настройки, включая [конфигурацию MCP-сервера](concepts_and_terminology/MCP_SERVERS.md), но он может не взаимодействовать чисто с оркестрационным ПО, которое существует сегодня. Для большинства workflow агенты на стороне хоста проще и надёжнее.

## Coasts vs Dev Containers

Coasts — это не dev containers, и это не одно и то же.

Dev containers обычно предназначены для монтирования IDE в единственное контейнеризованное рабочее пространство разработки. Coasts не имеют UI (headless) и оптимизированы как лёгкие окружения для параллельного использования агентами с worktrees — несколько изолированных, осведомлённых о worktrees сред выполнения, работающих бок о бок, с быстрым переключением checkout и средствами контроля изоляции среды выполнения для каждого инстанса.
