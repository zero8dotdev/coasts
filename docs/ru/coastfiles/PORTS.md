# Порты

Раздел `[ports]` объявляет, какими портами Coast управляет для проброса между вашими экземплярами Coast и хост-машиной. Необязательный раздел `[egress]` объявляет порты на хосте, к которым экземплярам Coast нужно обращаться наружу.

О том, как проброс портов работает во время выполнения — канонические vs динамические порты, переключение checkout, socat — см. [Ports](../concepts_and_terminology/PORTS.md) и [Checkout](../concepts_and_terminology/CHECKOUT.md).

## `[ports]`

Плоская карта `logical_name = port_number`. Каждая запись говорит Coast настраивать проброс порта для этого порта при запуске экземпляра Coast.

```toml
[ports]
web = 3000
api = 8080
postgres = 5432
```

Каждый экземпляр получает динамический порт (в верхнем диапазоне, всегда доступный) для каждого объявленного порта. Экземпляр [checked-out](../concepts_and_terminology/CHECKOUT.md) также получает канонический порт (число, которое вы объявили), проброшенный на хост.

Правила:

- Значения портов должны быть ненулевыми беззнаковыми 16-битными целыми числами (1-65535).
- Логические имена — произвольные строки, используемые как идентификаторы в `coast ports`, Coastguard и `primary_port`.

### Минимальный пример

```toml
[coast]
name = "my-app"
compose = "./docker-compose.yml"

[ports]
app = 3000
```

### Пример с несколькими сервисами

```toml
[ports]
web = 3000
api = 4000
backend = 8080
postgres = 5432
redis = 6379
```

## `primary_port`

Задаётся в секции `[coast]` (описано в [Project and Setup](PROJECT.md)); `primary_port` задаёт имя одного из ваших объявленных портов для быстрых ссылок и маршрутизации по поддоменам в [Coastguard](../concepts_and_terminology/COASTGUARD.md).

```toml
[coast]
name = "my-app"
primary_port = "web"

[ports]
web = 3000
api = 8080
```

Значение должно совпадать с ключом в `[ports]`. Подробнее см. [Primary Port and DNS](../concepts_and_terminology/PRIMARY_PORT_AND_DNS.md).

## `[egress]`

Объявляет порты на хосте, к которым экземплярам Coast нужно обращаться. Это направление, обратное `[ports]` — вместо проброса порта *из* Coast на хост, egress делает порт хоста доступным *изнутри* Coast.

```toml
[coast]
name = "my-app"
compose = "./docker-compose.yml"

[ports]
app = 48090

[egress]
host-api = 48080
```

Это полезно, когда ваши compose-сервисы внутри Coast должны обращаться к чему-то, запущенному напрямую на хост-машине (вне системы общих сервисов Coast).

Правила:

- Те же, что и для `[ports]`: значения должны быть ненулевыми беззнаковыми 16-битными целыми числами.
- Логические имена — произвольные идентификаторы.
