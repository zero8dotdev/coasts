# Checkout

Checkout управляет тем, какой экземпляр Coast владеет [каноническими портами](PORTS.md) вашего проекта. Когда вы выполняете checkout для Coast, `localhost:3000`, `localhost:5432` и любой другой канонический порт начинают напрямую указывать на этот экземпляр.

```bash
coast checkout dev-1
```

```text
Before checkout:
  localhost:3000  ──→  (nothing)
  localhost:5432  ──→  (nothing)

After checkout:
  localhost:3000  ──→  dev-1 web
  localhost:5432  ──→  dev-1 db
```

Переключение checkout происходит мгновенно — Coast завершает и заново запускает легковесные `socat`-форвардеры. Ни один контейнер не перезапускается.

```bash
coast checkout dev-2   # instant swap

# localhost:3000  ──→  dev-2 web
# localhost:5432  ──→  dev-2 db
```

## Linux Note

Динамические порты в Linux всегда работают без специальных привилегий.

С каноническими портами ниже `1024` ситуация иная. Если ваш Coastfile объявляет порты вроде `80` или `443`, Linux может заблокировать привязку этих портов командой `coast checkout`, пока вы не настроите хост. Обычно это исправляется так:

- увеличить `net.ipv4.ip_unprivileged_port_start`
- выдать capability на bind бинарнику или процессу форвардинга

Coast явно сообщает об этом, когда хост запрещает bind.

В WSL Coast использует checkout-мосты, опубликованные через Docker, чтобы браузеры и инструменты Windows могли обращаться к checked-out каноническим портам через `127.0.0.1`, аналогично сценариям Docker Desktop, таким как Sail.

## Do You Need to Check Out?

Не обязательно. У каждого запущенного Coast всегда есть собственные динамические порты, и вы можете обращаться к любому Coast через эти порты в любой момент, ничего не делая checkout.

```bash
coast ports dev-1

# SERVICE    CANONICAL  DYNAMIC
# ★ web      3000       62217
#   db       5432       55681
```

Вы можете открыть `localhost:62217` в браузере, чтобы попасть на веб-сервер dev-1, не делая checkout. Для многих сценариев этого вполне достаточно, и вы можете запускать сколько угодно Coast, ни разу не используя `coast checkout`.

## When Checkout Is Useful

Бывают ситуации, когда динамических портов недостаточно и нужны канонические порты:

- **Клиентские приложения, жестко привязанные к каноническим портам.** Если у вас есть клиент, работающий вне Coast — frontend dev server на вашем хосте, мобильное приложение на телефоне или десктопное приложение — и он ожидает `localhost:3000` или `localhost:8080`, менять номера портов повсюду непрактично. Checkout Coast дает вам реальные порты без изменения какой-либо конфигурации.

- **Webhook’и и callback URL.** Такие сервисы, как Stripe, GitHub или OAuth-провайдеры, отправляют callback-запросы на URL, который вы зарегистрировали — обычно что-то вроде `https://your-ngrok-tunnel.io`, перенаправляющее на `localhost:3000`. Если вы переключитесь на динамический порт, callback’и перестанут приходить. Checkout гарантирует, что канонический порт активен для Coast, который вы тестируете.

- **Инструменты для работы с базами данных, отладчики и интеграции IDE.** Многие GUI-клиенты (pgAdmin, DataGrip, TablePlus), отладчики и конфигурации запуска в IDE сохраняют профили подключения с конкретным портом. Checkout позволяет сохранить эти профили и просто переключать, какой Coast стоит за ними — без перенастройки цели подключения отладчика или соединения с базой данных при каждом переключении контекста.

## Releasing Checkout

Если вы хотите освободить канонические порты, не выполняя checkout другого Coast:

```bash
coast checkout --none
```

После этого ни один Coast не владеет каноническими портами. Все Coast остаются доступными через свои динамические порты.

## Only One at a Time

В каждый момент времени checkout может быть выполнен ровно для одного Coast. Если checkout сделан для `dev-1` и вы запускаете `coast checkout dev-2`, канонические порты мгновенно переключаются на `dev-2`. Разрыва нет — старые форвардеры завершаются, а новые запускаются в рамках той же операции.

```text
┌──────────────────────────────────────────────────┐
│  Your machine                                    │
│                                                  │
│  Canonical (checked-out Coast only):             │
│    localhost:3000 ──→ dev-2 web                  │
│    localhost:5432 ──→ dev-2 db                   │
│                                                  │
│  Dynamic (always available):                     │
│    localhost:62217 ──→ dev-1 web                 │
│    localhost:55681 ──→ dev-1 db                  │
│    localhost:63104 ──→ dev-2 web                 │
│    localhost:57220 ──→ dev-2 db                  │
└──────────────────────────────────────────────────┘
```

Динамические порты не затрагиваются checkout. Меняется только то, куда указывают канонические порты.
