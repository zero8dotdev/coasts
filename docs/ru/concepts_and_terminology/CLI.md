# Coast CLI

Coast CLI (`coast`) — это основной интерфейс командной строки для управления Coasts. Он намеренно сделан минималистичным: разбирает вашу команду, отправляет запрос в [`coastd`](DAEMON.md) и выводит структурированный результат обратно в ваш терминал.

## What You Use It For

Типичные рабочие процессы целиком выполняются из CLI:

```bash
coast build                                    # see Builds
coast run dev-1                                # see Coasts
coast assign dev-1 --worktree feature/oauth    # see Assign
coast ports dev-1                              # see Ports
coast checkout dev-1                           # see Checkout
coast ui                                       # see Coastguard
```

CLI также включает команды документации, полезные для людей и агентов:

```bash
coast docs
coast docs --path concepts_and_terminology/CHECKOUT.md
coast search-docs "canonical vs dynamic ports"
```

## Why It Exists Separately from the Daemon

Отделение CLI от демона даёт несколько важных преимуществ:

- Демон хранит состояние и поддерживает долгоживущие процессы.
- CLI остаётся быстрым, компонуемым и удобным для написания скриптов.
- Вы можете запускать разовые команды, не удерживая состояние терминала.
- Инструменты для агентов могут вызывать команды CLI предсказуемым, удобным для автоматизации способом.

## CLI vs Coastguard

Используйте тот интерфейс, который лучше подходит в данный момент:

- CLI рассчитан на полное операционное покрытие: всё, что можно сделать в Coastguard, должно быть возможно и из CLI.
- Рассматривайте CLI как интерфейс для автоматизации — скрипты, агентные рабочие процессы, задания CI и кастомные инструменты разработчика.
- Рассматривайте [Coastguard](COASTGUARD.md) как интерфейс для человека — визуальная проверка, интерактивная отладка и операционная наблюдаемость.

Оба взаимодействуют с одним и тем же демоном, поэтому работают с одним и тем же базовым состоянием проекта.
