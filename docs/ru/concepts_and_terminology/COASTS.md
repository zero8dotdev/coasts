# Побережья

Побережье (Coast) — это самодостаточная среда выполнения вашего проекта. Она запускается внутри [контейнера Docker-in-Docker](RUNTIMES_AND_SERVICES.md), и несколько сервисов (ваш веб-сервер, база данных, кэш и т. д.) могут работать внутри одного экземпляра Coast.

```text
┌─── Coast: dev-1 (branch: feature/oauth) ──────────────┐
│                                                       │
│   ┌─────────┐   ┌──────────┐   ┌─────────┐            │
│   │   web   │   │ postgres │   │  redis  │            │
│   │  :3000  │   │  :5432   │   │  :6379  │            │
│   └─────────┘   └──────────┘   └─────────┘            │
│                                                       │
│   dynamic ports: 62217, 55681, 56905                  │
└───────────────────────────────────────────────────────┘

┌─── Coast: dev-2 (branch: feature/billing) ────────────┐
│                                                       │
│   ┌─────────┐   ┌──────────┐   ┌─────────┐            │
│   │   web   │   │ postgres │   │  redis  │            │
│   │  :3000  │   │  :5432   │   │  :6379  │            │
│   └─────────┘   └──────────┘   └─────────┘            │
│                                                       │
│   dynamic ports: 63104, 57220, 58412                  │
└───────────────────────────────────────────────────────┘
```

Каждое Побережье открывает для хост-машины собственный набор [динамических портов](PORTS.md), а значит вы можете получить доступ к любому запущенному Побережью в любой момент, независимо от того, что ещё запущено.

Когда вы [переключаетесь](CHECKOUT.md) на Побережье, на него маппятся канонические порты проекта — поэтому `localhost:3000` попадает в переключённое Побережье, а не в динамический порт.

```text
coast checkout dev-1

localhost:3000  ──→  dev-1 web
localhost:5432  ──→  dev-1 postgres
localhost:6379  ──→  dev-1 redis

coast checkout dev-2   (instant swap)

localhost:3000  ──→  dev-2 web
localhost:5432  ──→  dev-2 postgres
localhost:6379  ──→  dev-2 redis
```

Обычно Побережье [назначается конкретному worktree](ASSIGN.md). Так вы запускаете несколько worktree одного и того же проекта параллельно без конфликтов портов или коллизий томов.

Вы сами решаете, когда поднимать и когда останавливать Побережья. Вероятно, вы не захотите держать одновременно запущенными 20 Побережий ресурсоёмкого проекта, но каждому своё.
