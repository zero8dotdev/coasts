# Демон Coast

Демон Coast (`coastd`) — это долгоживущий локальный процесс, который выполняет фактическую работу по оркестрации. [CLI](CLI.md) и [Coastguard](COASTGUARD.md) являются клиентами; `coastd` — это плоскость управления, стоящая за ними.

## Архитектура в общих чертах

```text
coast CLI (автоматизация) -----+
                               +--> демон coastd
Coastguard UI (человек) -------+         |
                                         +--> Coasts
                                         +--> Ports
                                         +--> State
```

CLI отправляет запросы в `coastd` через локальный Unix-сокет; Coastguard подключается по WebSocket. Демон применяет изменения к состоянию выполнения.

## Что он делает

`coastd` обрабатывает операции, которым требуется постоянное состояние и фоновая координация:

- Отслеживает экземпляры Coast, сборки и общие сервисы.
- Создаёт, запускает, останавливает и удаляет среды выполнения Coast.
- Применяет операции assign/unassign/checkout.
- Управляет каноническим и динамическим [пробросом портов](PORTS.md).
- Передаёт потоком [логи](LOGS.md), статус и события выполнения клиентам CLI и UI.

Короче говоря: если вы выполняете `coast run`, `coast assign`, `coast checkout` или `coast ls`, демон — это компонент, который выполняет работу.

## Как он запускается

Вы можете запустить демон двумя распространёнными способами:

```bash
# Register daemon auto-start at login (recommended)
coast daemon install

# Manual start mode
coast daemon start
```

Если вы пропустите установку демона, вам нужно будет запускать его самостоятельно в каждой сессии перед использованием команд Coast.

## Сообщение об ошибках

Если у вас возникли проблемы, пожалуйста, приложите логи демона `coastd` при отправке отчёта об ошибке. Логи содержат контекст, необходимый для диагностики большинства проблем:

```bash
coast daemon logs
```
