# Порты

Coast управляет двумя видами сопоставления портов для каждого сервиса в экземпляре Coast: каноническими портами и динамическими портами.

## Канонические порты

Это порты, на которых ваш проект обычно работает — те, что указаны в вашем `docker-compose.yml` или локальной конфигурации для разработки. Например, `3000` для веб-сервера, `5432` для Postgres.

Только один Coast может одновременно иметь канонические порты. Их получает тот Coast, который [выбран](CHECKOUT.md).

```text
coast checkout dev-1

localhost:3000  ──→  dev-1
localhost:5432  ──→  dev-1
```

Это означает, что ваш браузер, API-клиенты, инструменты для работы с базой данных и наборы тестов работают точно так же, как обычно, — менять номера портов не нужно.

В Linux для канонических портов ниже `1024` может потребоваться настройка хоста, прежде чем [`coast checkout`](CHECKOUT.md) сможет привязать их. У динамических портов этого ограничения нет.

## Динамические порты

Каждый запущенный Coast всегда получает собственный набор динамических портов в верхнем диапазоне (49152–65535). Они назначаются автоматически и всегда доступны независимо от того, какой Coast выбран.

```text
coast ports dev-1

# SERVICE    CANONICAL  DYNAMIC
# ★ web      3000       62217
#   db       5432       55681

coast ports dev-2

# SERVICE    CANONICAL  DYNAMIC
# ★ web      3000       63104
#   db       5432       57220
```

Динамические порты позволяют вам заглянуть в любой Coast, не переключая его. Вы можете открыть `localhost:63104`, чтобы обратиться к веб-серверу dev-2, пока dev-1 выбран на канонических портах.

## Как они работают вместе

```text
┌──────────────────────────────────────────────────┐
│  Your machine                                    │
│                                                  │
│  Canonical (checked-out Coast only):             │
│    localhost:3000 ──→ dev-1 web                  │
│    localhost:5432 ──→ dev-1 db                   │
│                                                  │
│  Dynamic (always available):                     │
│    localhost:62217 ──→ dev-1 web                 │
│    localhost:55681 ──→ dev-1 db                  │
│    localhost:63104 ──→ dev-2 web                 │
│    localhost:57220 ──→ dev-2 db                  │
└──────────────────────────────────────────────────┘
```

Переключение [checkout](CHECKOUT.md) происходит мгновенно — Coast завершает и заново запускает легковесные `socat`-перенаправители. Ни один контейнер не перезапускается.

См. также [Primary Port & DNS](PRIMARY_PORT_AND_DNS.md) о быстрых ссылках, маршрутизации по поддоменам и шаблонах URL.
