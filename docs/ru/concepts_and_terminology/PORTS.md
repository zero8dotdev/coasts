# Порты

Coast управляет двумя видами сопоставления портов для каждого сервиса в экземпляре Coast: каноническими портами и динамическими портами.

## Канонические порты

Это порты, на которых ваш проект обычно работает — те, что указаны в вашем `docker-compose.yml` или локальной dev-конфигурации. Например, `3000` для веб-сервера, `5432` для Postgres.

Одновременно канонические порты может иметь только один Coast. Их получает тот Coast, который [checked out](CHECKOUT.md).

```text
coast checkout dev-1

localhost:3000  ──→  dev-1
localhost:5432  ──→  dev-1
```

Это означает, что ваш браузер, API-клиенты, инструменты для работы с базой данных и тестовые наборы работают ровно так же, как обычно — менять номера портов не нужно.

## Динамические порты

Каждый запущенный Coast всегда получает собственный набор динамических портов в высоком диапазоне (49152–65535). Они назначаются автоматически и всегда доступны независимо от того, какой Coast checked out.

```text
coast ports dev-1

# SERVICE    CANONICAL  DYNAMIC
# ★ web      3000       62217
#   db       5432       55681

coast ports dev-2

# SERVICE    CANONICAL  DYNAMIC
# ★ web      3000       63104
#   db       5432       57220
```

Динамические порты позволяют заглянуть в любой Coast, не выполняя checkout. Вы можете открыть `localhost:63104`, чтобы попасть на веб-сервер dev-2, пока dev-1 checked out на канонических портах.

## Как они работают вместе

```text
┌──────────────────────────────────────────────────┐
│  Ваша машина                                     │
│                                                  │
│  Канонические (только checked-out Coast):        │
│    localhost:3000 ──→ dev-1 web                  │
│    localhost:5432 ──→ dev-1 db                   │
│                                                  │
│  Динамические (всегда доступны):                 │
│    localhost:62217 ──→ dev-1 web                 │
│    localhost:55681 ──→ dev-1 db                  │
│    localhost:63104 ──→ dev-2 web                 │
│    localhost:57220 ──→ dev-2 db                  │
└──────────────────────────────────────────────────┘
```

Переключение [checkout](CHECKOUT.md) происходит мгновенно — Coast завершает и заново поднимает лёгкие форвардеры `socat`. Никакие контейнеры не перезапускаются.

См. также [Primary Port & DNS](PRIMARY_PORT_AND_DNS.md) про быстрые ссылки, маршрутизацию по поддоменам и шаблоны URL.
