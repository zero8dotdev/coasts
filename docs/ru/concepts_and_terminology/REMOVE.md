# Удаление

`coast rm` полностью удаляет экземпляр Coast. Она останавливает экземпляр, если
он запущен, удаляет контейнер DinD, удаляет изолированные тома, освобождает
порты, удаляет agent shells и удаляет экземпляр из состояния.

```bash
coast rm dev-1
```

В большинстве повседневных рабочих процессов `coast rm` не нужна. Если вы
просто хотите, чтобы Coast запускал другой код или владел каноническими
портами, используйте [Assign and Unassign](ASSIGN.md) или [Checkout](CHECKOUT.md)
вместо этого. Обращайтесь к `coast rm`, когда хотите остановить Coasts,
освободить состояние среды выполнения для конкретного экземпляра или
пересоздать экземпляр с нуля после пересборки вашего Coastfile или сборки.

## Что происходит

`coast rm` выполняет пять этапов:

1. **Проверка и поиск** — ищет экземпляр в состоянии. Если запись в состоянии
   исчезла, но всё ещё существует зависший контейнер с ожидаемым именем,
   `coast rm` тоже очистит его.
2. **Остановка при необходимости** — если экземпляр находится в состоянии
   `Running` или `CheckedOut`, Coast сначала останавливает внутренний compose
   stack и контейнер DinD.
3. **Удаление артефактов среды выполнения** — удаляет контейнер Coast и
   удаляет изолированные тома для этого экземпляра.
4. **Очистка состояния хоста** — завершает оставшиеся перенаправители портов,
   освобождает порты, удаляет agent shells и удаляет запись экземпляра из базы
   данных состояния.
5. **Сохранение общих данных** — общие тома сервисов и общие данные сервисов
   остаются нетронутыми.

## Использование CLI

```text
coast rm <name>
coast rm --all
```

| Flag | Description |
|------|-------------|
| `<name>` | Удалить один экземпляр по имени |
| `--all` | Удалить все экземпляры для текущего проекта |

`coast rm --all` определяет текущий проект, перечисляет его экземпляры и
удаляет их по одному. Если экземпляров нет, команда корректно завершится.

## Общие сервисы и сборки

- `coast rm` **не** удаляет данные общих сервисов.
- Используйте `coast shared-services rm <service>`, если вы также хотите
  удалить общий сервис и его данные.
- Используйте `coast rm-build`, если хотите удалить артефакты сборки после
  остановки экземпляров.

## Когда это использовать

- после пересборки вашего Coastfile или создания новой сборки, когда нужен
  новый экземпляр
- когда вы хотите остановить Coasts и освободить состояние контейнеров и томов
  для конкретного экземпляра
- когда экземпляр находится в нерабочем состоянии, и начать заново проще, чем
  отлаживать его на месте
