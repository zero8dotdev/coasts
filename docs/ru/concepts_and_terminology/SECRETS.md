# Секреты и экстракторы

Секреты — это значения, извлечённые с вашей хост-машины и внедряемые в контейнеры Coast как переменные окружения или файлы. Coast извлекает секреты во время сборки, шифрует их при хранении в локальном хранилище ключей и внедряет их при создании экземпляра Coast.

## Типы внедрения

У каждого секрета есть цель `inject`, которая управляет тем, как он доставляется в контейнер Coast:

- `env:VAR_NAME` — внедряется как переменная окружения.
- `file:/path/in/container` — монтируется как файл внутри контейнера.

```toml
[secrets.api_key]
extractor = "env"
var = "API_KEY"
inject = "env:API_KEY"

[secrets.credentials]
extractor = "file"
path = "~/.config/my-app/credentials.json"
inject = "file:/run/secrets/credentials.json"
```

## Встроенные экстракторы

### env

Читает переменную окружения хоста. Это самый распространённый и самый простой экстрактор. Если у вас уже есть секреты в виде переменных окружения на хосте — из файлов `.env`, `direnv`, профилей shell или любого другого источника — просто пробросьте их в Coast.

```toml
[secrets.db_password]
extractor = "env"
var = "DB_PASSWORD"
inject = "env:DATABASE_PASSWORD"
```

Большинству проектов достаточно одного экстрактора `env`.

### file

Читает файл из файловой системы хоста. Поддерживает разворачивание `~` для путей к домашнему каталогу. Подходит для SSH-ключей, TLS-сертификатов и JSON-файлов с учётными данными.

```toml
[secrets.ssh_key]
extractor = "file"
path = "~/.ssh/id_ed25519"
inject = "file:/run/secrets/ssh_key"
```

### command

Запускает команду оболочки и захватывает stdout как значение секрета. Команда выполняется через `sh -c`, поэтому работают пайпы, перенаправления и подстановка переменных. Это полезно для получения секретов из 1Password CLI, HashiCorp Vault или любого динамического источника.

```toml
[secrets.op_token]
extractor = "command"
run = "op read 'op://vault/db/password'"
inject = "env:DATABASE_PASSWORD"
```

Вы также можете использовать `command`, чтобы преобразовывать или извлекать конкретные поля из локальных конфигурационных файлов:

```toml
[secrets.claude_config]
extractor = "command"
run = 'python3 -c "import json; print(json.dumps({\"key\": \"value\"}))"'
inject = "file:/root/.claude.json"
```

### keychain

Псевдоним для `macos-keychain`. Читает элемент типа generic password из Связки ключей macOS.

```toml
[secrets.claude_credentials]
extractor = "keychain"
service = "Claude Code-credentials"
inject = "file:/root/.claude/.credentials.json"
```

Экстрактор keychain часто не нужен. Если вы можете получить то же значение через переменную окружения или файл, предпочитайте эти более простые подходы. Извлечение из Keychain полезно, когда секрет существует только в Связке ключей macOS и его не так просто экспортировать — например, специфичные для приложения учётные данные, сохранённые сторонними инструментами, которые пишут напрямую в Keychain.

Параметр `account` необязателен и по умолчанию равен вашему имени пользователя macOS.

Этот экстрактор доступен только на macOS. Упоминание его на других платформах приведёт к понятной ошибке во время сборки.

## Пользовательские экстракторы

Если ни один из встроенных экстракторов не подходит под ваш рабочий процесс, Coast переходит к поиску исполняемого файла с именем `coast-extractor-{name}` в вашем PATH. Исполняемый файл получает параметры экстрактора как JSON через stdin и должен вывести значение секрета в stdout.

```toml
[secrets.vault_token]
extractor = "vault"
path = "secret/data/token"
inject = "env:VAULT_TOKEN"
```

Coast вызовет `coast-extractor-vault`, передав `{"path": "secret/data/token"}` в stdin. Код выхода 0 означает успех; ненулевой — неудачу (stderr включается в сообщение об ошибке).

## Внедрение несекретных значений

Раздел `[inject]` пробрасывает переменные окружения и файлы хоста в Coast, не считая их секретами. Эти значения не шифруются — они передаются напрямую.

```toml
[inject]
env = ["NODE_ENV", "DEBUG"]
files = ["~/.gitconfig", "~/.npmrc"]
```

Используйте `[inject]` для конфигурации, которая не является чувствительной. Используйте `[secrets]` для всего, что должно быть зашифровано при хранении.

## Секреты не сохраняются в сборке

Секреты извлекаются во время сборки, но не встраиваются в артефакт сборки coast. Они внедряются при создании экземпляра Coast с `coast run`. Это означает, что вы можете делиться артефактами сборки, не раскрывая секреты.

Секреты можно повторно внедрять во время выполнения без пересборки. В интерфейсе [Coastguard](COASTGUARD.md) используйте действие **Re-run Secrets** на вкладке Secrets. Из CLI используйте [`coast build --refresh`](BUILDS.md), чтобы повторно извлечь и обновить секреты.

## TTL и повторное извлечение

Секреты могут иметь необязательное поле `ttl` (time-to-live). Когда срок действия секрета истекает, `coast build --refresh` повторно извлечёт его из источника.

```toml
[secrets.short_lived_token]
extractor = "command"
run = "generate-token --ttl 1h"
inject = "env:AUTH_TOKEN"
ttl = "1h"
```

## Шифрование при хранении

Все извлечённые секреты шифруются с помощью AES-256-GCM в локальном хранилище ключей. Ключ шифрования хранится в Связке ключей macOS на macOS или в файле с правами 0600 на Linux.
