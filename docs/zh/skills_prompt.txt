# Coast Runtime

此项目使用 Coasts（容器化主机）来提供隔离的开发环境。
你的代码编辑会自动在正在运行的 Coast 内可见——文件系统
在主机和容器之间共享。

## Orientation

在运行任何运行时命令之前，先发现哪个 Coast 实例与你的
当前工作目录匹配:

```
coast lookup
```

这会打印实例名称、端口、URL 和示例命令。对所有后续命令，
都使用输出中的实例名称。

如果你需要更深入了解 Coasts 的工作方式，请阅读以下文档:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

使用 `coast exec` 在 Coast 内运行命令。shell 会从工作区
根目录（即 Coastfile 所在位置）启动。请先 cd 到你的目标目录:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

示例:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

检查服务状态:

```
coast ps <instance>
```

读取服务日志:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

如果你遇到错误或不熟悉的行为，请搜索 Coast 文档:

```
coast search-docs "error message or description"
```

这会使用语义搜索——用自然语言描述问题，它会
找到相关文档。

## Worktree Awareness

当你开始在某个 worktree 中工作时——无论是你创建的，还是像
Codex、Conductor 或 T3 Code 这样的工具为你创建的——请检查是否
已经分配了一个 Coast 实例:

```
coast lookup
```

如果 `coast lookup` 找到了一个实例，请将其用于所有运行时命令。

如果它没有返回任何实例，请检查当前正在运行的内容:

```
coast ls
```

然后询问用户他们更喜欢哪个选项:

**选项 1 —— 创建一个新的 Coast 并分配此 worktree:**
```
coast run <new-name>
coast assign <new-name> -w <worktree>
```

**选项 2 —— 将现有 Coast 重新分配给此 worktree:**
```
coast assign <existing-name> -w <worktree>
```

**选项 3 —— 完全跳过 Coast:**
在没有运行时环境的情况下继续。你可以编辑文件，但不能在容器内运行测试、
构建或服务。

`<worktree>` 的值是分支名称（运行 `git branch --show-current`）或
`coast ls` 中显示的 worktree 标识符。创建或重新分配前始终先询问用户——不要自动执行。

## Rules

- 在会话中的第一个运行时命令之前，始终运行 `coast lookup`。
- 不要直接在主机上运行服务。所有运行时任务都使用 `coast exec`。
- 在主机上的文件编辑会立即在 Coast 内可见。你不需要
  在编辑后复制文件或重新构建。
- 如果 `coast lookup` 没有返回任何实例，Coast 可能没有运行。请遵循
  上面的 worktree awareness 流程，与用户一起解决此问题。
