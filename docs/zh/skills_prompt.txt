# Coast Runtime

此项目使用 Coasts——用于运行服务、测试和其他运行时命令的容器化运行时。主机与容器之间共享文件系统，因此任一侧的文件编辑都会立即对双方可见。

## Discovery

在会话中的第一个运行时命令之前，运行:

  coast lookup

这会打印实例名称、端口和示例命令。对所有后续命令，都使用输出中的实例名称。

## What runs where

文件系统是共享的，因此只有在需要容器运行时的情况下才使用 `coast exec`（数据库、服务、集成测试）。其他所有内容都直接在主机上运行。

在以下情况下使用 `coast exec`:
- 需要运行中服务的测试（与服务或数据库集成的单元测试、集成测试、API 测试）
- 服务重启或 compose 操作
- 任何与数据库、缓存或其他容器服务交互的内容

直接在主机上运行:
- Lint、类型检查、格式化
- Git 操作
- Playwright 和浏览器测试
- 安装主机侧依赖（npm install、pip install）
- 文件搜索、代码生成、静态分析

示例:

  coast exec <instance> -- sh -c "cd <dir> && npm test"    # needs DB
  npm run lint                                              # host is fine
  npx playwright test                                       # host is fine

## Runtime feedback

  coast ps <instance>
  coast logs <instance> --service <service>
  coast logs <instance> --service <service> --tail 50

## Creating and assigning Coasts

如果 `coast lookup` 没有返回匹配项，运行 `coast ls` 查看已有内容。

如果此项目已有一个未分配的 Coast 正在运行，优先将你的 worktree 分配给它，而不是创建新的:

  coast assign <existing> -w <worktree>

已被占用的 Coast 也可以使用 `coast assign` 重新分配，但要先与用户确认，因为这会扰乱当前槽位。

如果当前没有运行中的 Coast，请在创建前先询问用户——Coasts 可能会占用大量内存:

  coast run <name> -w <worktree>

项目必须先构建，才能创建实例。如果 `coast run` 因为没有现有构建而失败，先运行 `coast build`。

## Coastfile setup

如果项目还没有 Coastfile，或者你需要修改 Coastfile，请先阅读 Coastfile 文档:

  coast docs --path coastfiles/README.md

## When confused

在猜测 Coast 行为之前，先查看文档:

  coast docs                                     # 列出所有文档页面
  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast search-docs "your question here"         # 语义搜索

## Rules

- 在会话中的第一个运行时命令之前，始终运行 `coast lookup`。
- 仅对需要容器运行时的内容使用 `coast exec`。
- 直接在主机上运行 lint、类型检查、格式化和 git。
- 在猜测 Coast 行为之前，使用 `coast docs` 或 `coast search-docs`。
- 当项目要求使用 Coast 时，不要直接在主机上运行服务。

# Coasts

直接使用 Coast CLI。不要添加包装器。

## Orient Yourself

先浏览 CLI 和文档:

  coast                                             # 查看所有可用命令
  coast docs                                        # 列出所有文档页面
  coast search-docs "your question"                 # 语义搜索

如果对 Coast 的行为有任何不清楚之处，请先阅读文档，不要猜测:

  coast docs --path concepts_and_terminology/RUN.md
  coast docs --path concepts_and_terminology/BUILDS.md
  coast docs --path concepts_and_terminology/ASSIGN.md
  coast docs --path concepts_and_terminology/PORTS.md
  coast docs --path coastfiles/README.md

## Quick Start

将请求路由到以下模式之一:

1. **Use Coast** —— 运行 `coast lookup`，然后对匹配的实例使用 `coast exec`、`coast ps` 或 `coast logs`。
2. **Create or Assign** —— 运行 `coast ls`，然后使用 `coast run` 创建新的 Coast，或使用 `coast assign` 重新指向现有 Coast。
3. **Open UI** —— 运行 `coast ui`。

## What Runs Where

主机与 Coast 共享文件系统。仅在需要容器内运行中的服务时才使用 `coast exec`。

**在以下情况下使用 `coast exec`:**
- 集成测试、API 测试，以及任何需要数据库或服务的内容
- 服务重启、compose 操作
- 与仅存在于容器中的进程交互的命令

**在主机上运行:**
- Lint (`eslint`, `rubocop`, `golangci-lint`)
- 类型检查 (`tsc --noEmit`, `go vet`)
- 格式化 (`prettier`, `gofmt`)
- Git 操作
- Playwright 和浏览器测试
- 静态分析、代码生成
- 安装包 (`npm install`, `pip install`)

## Create and Assign

当 `coast lookup` 没有返回匹配项时:

1. 运行 `coast ls` 查看可用槽位。
2. 优先使用 `coast run <name> -w <worktree>`，一步完成创建和分配。
3. 如果还没有构建，先运行 `coast build`。
4. 创建后，重新运行 `coast lookup` 进行确认。

当你想将现有 Coast 切换到不同的 worktree 时:

  coast assign <name> -w <worktree>

这也适用于已经分配或已 checkout 的 Coast，但在重新分配已占用槽位前，先询问用户。

## Coastfile Setup

如果项目需要新的或修改后的 Coastfile，请先阅读文档:

  coast docs --path coastfiles/README.md

Coastfile 文档涵盖 compose 配置、端口、卷、密钥、共享服务、裸服务和继承。

## Safety Rules

- 在采取行动前运行 `coast lookup`，并在任何拓扑变更后再次运行。
- 如果 `coast assign`、`coast unassign` 或 `coast checkout` 会扰乱现有槽位，先征求同意。
- 除非用户明确希望重新分配现有槽位，否则优先创建新的 Coast，而不是复用已 checkout 或已分配的 Coast。
- 在猜测之前，使用 `coast docs` 或 `coast search-docs`。
